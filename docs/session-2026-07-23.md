## The recovery story (read before rebuilding anything "already built")
The BoxVacay treatment was not in the repo, its git history, or any Vercel deployment. It lived
only in ~/Downloads as `nathanjsong-modals-preview-v8/v8_1/v9.html` — previews from a 2026-07-20
session that were never committed. Preview files in Downloads are the ONLY copy of uncommitted
work; check there first. Ported verbatim from that v9 with three deliberate changes: `soon:false`
(the site is live, so the chip wakes to ember + "Explore Camps" — the awake state its data
already planned), the dock icon takes the ember color, and `.bv-horizon` moves up on phones
(the ridge drew through the eyebrow text — present in the original preview too).

## Decisions, including rejected ones
- "Side Projects" tab + a per-modal hobby line — rejected by Jay as overboard. Standing rule:
  positioning gets ONE quiet mention; here it is the dock tab "Personal Portfolio".
- "Case Study · No. I" eyebrows — replaced. "Case study" implies client work, and Roman-numeral
  numbering added nothing next to real dates. Date-only plates now.
- `soon:true` machinery kept in the code: a future project can launch greyed as "Coming soon"
  (one flag) and wake later.
- No chip-generation refactor. Adding a project stays copy-one-chip + one DATA entry; recipe
  comments sit above the dock markup and the DATA object.

## Traps
- Bridge git leaves stale `.git/index.lock` / `HEAD.lock` it cannot delete. Move them into
  `_to_delete/`, and `rm -f` them before any Mac-side git command.
- The dock hides until ~320px of scroll; screenshots need a scroll first.
- Base `.dock` max-width is calc(100vw - 440px) so a growing chip row never reaches the chat
  button; below 1080px it widens to full width because the button rides above the dock there.
